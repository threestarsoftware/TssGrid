TssGrid 機能比較

TssGrid の立ち位置は MIT（無料・商用可）× ノービルド × 日本語IME堅牢 × 業務フォーム特化。 「Excelを丸ごとブラウザに」ではなく、帳票・マスタ入力・勤怠・在庫のような日本語の入力業務を、軽く・確実にがねらいです。 下表は主要ライブラリとの概況比較と、他社が強い領域（＝TssGrid の伸びしろ）を正直にまとめたものです。

他社が強い領域 → TssGrid の今後（伸びしろ）

「無料公開して終わり」ではなく、ここを着実に埋めていくのが方針です。正直に並べることが、次のバージョンの地図になります。

  • 仮想スクロール（AG Grid / SpreadJS が強い） → 対応済み（virtual.html・virtual:true でopt-in）。可視窓だけ描画する窓方式で、実測10万行＝初期45ms。フィルタの基盤とも共用。割り切り: 全機能フル対応の「100点」ではなく、まだ一部制約あり（インデックスマッパ整備は継続）。
  • セル結合 merge → 対応済み。データセルの横/縦/矩形結合（merge.html・plugins/tss-merge.js）＝アンカー=左上が値/編集/選択、Undo対応・行列の挿入削除で座標追従。ヘッダ結合（nestedHeaders）も別途あり。割り切り: ソートは結合中ロック・縦結合中は行移動不可（分断回避）。
  • フィルタUI（各社） → サンプルで対応済み（filter.html：マスタ保持＋setData＋編集同期）。本体は太らせない方針。完全な非破壊フィルタ＋構造変更両立は行インデックスマッパとセット（将来）。
  • 条件付き書式 → 対応済み（cellClass で値/行を見てクラス付与＋編集で即追従。見た目は業務側CSS）。Excel風の「ルール設定UI」は将来。
  • 行固定 frozen rows → 対応済み（frozenRows。列固定 frozenCols と併用で四隅固定）。
  • xlsx 入出力（SpreadJS が完全） → CSV出力は対応済み。xlsx は外部ライブラリ連携を想定（本体非搭載の方針）。
  • モバイル／タッチ（AG Grid / SpreadJS） → 選択ハンドル・長押しメニュー等の参考を収集済み。pointer イベント一般化で対応予定。
  • 数式（Handsontable / SpreadJS） → あえて非搭載の方針。「Excelを再現」ではなく「入力業務を軽く確実に」に集中。必要なら計算は業務コード側で。

用途別おすすめ（正直版）

TssGrid ── 軽量・日本語IME堅牢。帳票・マスタ・勤怠・在庫の入力画面を、ビルド無しでサッと。MITで商用も無料。
SpreadJS ── Excel互換を丸ごと（数式・グラフ・ピボット・xlsx）。予算があり「ブラウザ版Excel」が要るなら。国産サポート。
AG Grid ── 超大量データ・サーバサイド・エンタープライズ。仮想スクロールとデータグリッドの王。
Handsontable ── スプレッドシート機能の全部入り（数式・結合・条件付き書式）。商用予算が取れる場合の定番。
jspreadsheet CE ── 軽量スプレッドシート。数式も使える。ライセンス条件が許容できれば手軽。
Tabulator ── MIT無料・ノービルドの多機能テーブル。仮想スクロール・ヘッダフィルタ・グルーピング・モバイルが強く、編集も一通り。TssGrid に一番近い無料勢。
選び方の軸 ── 「大量データ／Excel互換」は商用勢、「多機能な無料テーブル」は Tabulator、「日本語の入力業務を、極小・ビルド無しで」なら TssGrid（IME堅牢・全角半角・休日カレンダー・セル結合が標準）。
